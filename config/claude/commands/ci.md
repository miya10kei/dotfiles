## タスク

現在のPRのCI状態を確認し、失敗している場合はエラーを修正する

## 入力

ユーザーからの指示： $ARGUMENTS

## 手順

### 1. CI状態確認

```bash
gh pr checks
```

### 2. 失敗時の対応

1. 失敗したジョブのログを取得
   ```bash
   gh run view <run-id> --log-failed
   ```

2. エラー内容を解析
   - lint エラー
   - 型エラー
   - テスト失敗
   - ビルドエラー

3. エラーを修正

4. 修正をコミット & プッシュ
   - コミットメッセージ: `🐛 fix: <エラー内容>`

### 3. 成功確認

```bash
gh pr checks
```

## よくあるエラーと対処

| エラー種別 | 対処 |
|-----------|------|
| ESLint | 指摘箇所を修正、または `eslint --fix` |
| TypeScript | 型エラーを修正 |
| テスト失敗 | テストコードまたは実装を修正 |
| フォーマット | `prettier --write` または該当フォーマッタ実行 |

## ルール

- 修正は最小限に留める
- 無関係な変更を含めない
- ユーザーからの指示がある場合は、その指示を最優先とすること
