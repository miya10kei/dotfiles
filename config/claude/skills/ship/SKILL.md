# Ship Workflow

コード変更を本番にshipするための全ワークフローを実行します。

## 前提条件

以下のいずれかの状態であること:
- ブランチ作成前で、これから変更を行う
- ブランチ作成前で、既に変更がローカルに存在する
- 既にフィーチャーブランチで作業中

## ワークフロー

### Step 1: ブランチ作成

既にフィーチャーブランチにいる場合はスキップ

/branch を実行

### Step 2: コミット

/commit を実行

### Step 3: PR作成

/pr を実行

### Step 4: CI監視 & 修正

/ci_check を実行

失敗時は修正して再度 /ci_check を実行（成功するまで繰り返し、最大5回）

5回失敗したらユーザーに報告して中断

### Step 5: マージ確認

/merge_wait を実行

### Step 6: クリーンアップ

/branch_cleanup を実行

## 完了報告

ワークフロー完了後、以下を報告:

- PR URL
- マージされたコミット
- 削除したブランチ名
