## タスク

ユーザーがPull Request のレビューをおこなうので、レビューのガイドをおこなうこと。

## 入力

PR番号： $ARGUMENTS

## タスクの詳細

* Pull Request を読み込んで変更内容を理解すること
  * PR番号が指定された場合は、そのPR番号のPull Requestを読み込むこと
  * PR番号が指定されなかった場合は、ブランチに紐づくPull Requestを読み込むこと
* 変更内容をユーザーが理解するのに最適となるようなファイルの閲覧順番を提示すること
  * ユーザーは変更点を処理の流れにそって理解することを好むので考慮すること
  * Configなどの共通部分の変更はそれを使用する箇所より前にレビューできるようにすること
* ユーザーが提示した閲覧順番を承認したら、1ファイルずつユーザーと共にレビューを開始すること
* レビューを開始したら、ファイルパス、変更点の概要、あなたのレビュー結果、注意してレビューすべき点を提示すること
* ユーザーから「次」や「next」などの命令を受け取ったら、閲覧順番にそって次に見るべきファイルを提示すること
